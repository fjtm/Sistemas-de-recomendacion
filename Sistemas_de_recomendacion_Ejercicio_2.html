Sistema de recomendación por filtro colaborativo ALS (Alternating Least Squares)¶

Para crear un sistema de recomendacin basado en filtros colaborativos utilizaremos la funcin ALS proporcionada por PYSPARK.ML. El funcionamiento de este mtodo se basa en la factorizacin de matrices y se puede resumir en la siguiente imgen:

Los hiperparmetros principales del modelo ALS en pyspark son:

  • regParam : Constante de regularizacin del modelo.

  • rank : Nmero de factores latentes a considerar.

  • maxIter : Nmero de iteraciones que realizar el algortmo.

  • numUserBlocks : Nmero de bloques de usuarios.

  • numItemBlocks : Nmero de bloques de items.

    El modelo tambin cuenta con tres parmetros que toman como valores las columnas de usuarios, items y ratings.

    Por ltimo destacar el parmetro coldStartStrategy, el cual evita que el modelo falle al encontrarse con un usuario nuevo.

    • En nuestro caso no disponemos de ratings propiamente dichos, sino que tenemos el nmero de reproducciones de cata artista por cada usuario. De esta manera, es razonable suponer que aquellos msicos con un mayor nmero de reproducciones tendrn un mayor rating (aunque no tiene porqu ser as). De esta manera utilizaremos el nmero de reproducciones como escala de ratings.

    En un primer lugar se creo un sistema de recomendacin ALS con el nmero de reproducciones sin modificar. Si se hace as, tenemos el problema de que estos valores estn muy esparcidos en orden de magnitud. Y as, nos enfrentamos a dos situaciones:

  • Dejamos que el modelo realice un gran overfit con valores razonables en los parmetros. Lo cual repercute en que aunque las recomendaciones pueden ser razonables hacemos el modelo tremendamente rgido.

  • Nos vemos obligados a utilizar valores de regParam enormes (superiores a 100), para controlar el overfit haciendo el modelo ms flexible, pero con unos resultados nefastos.

    Para controlar este problema hay que escalar el nmero de reproducciones que se encuentra en un rango aproximado de 1-60000. Una manera de hacerlo es aplicando el logaritmo al nmero de reproducciones, que aunque no es la mejor manera de escalar nos permite preservar el orden. Destacar que podemos aplicar el logaritmo puesto que el menor nmero de reproducciones es 1.

Se han probado combinaciones del resto de los parmetros dentro de las posibilidades:

  • Para maxIter debido a la falta de memora solamente podiamos originalmente realizar 5 iteraciones. Finalmente decid elegir 10 iteraciones debido a que con 20 tardaba mucho ms y los resultado eran similares a los obtenidos con 10.

  • Con respecto a numUserBlocks y numItemBlocks he explorado menos estos hiperparmetros debido al enorme coste computacional que requera modificarlos en un rango importante. Los pocos cambios que he probado no han resultado ser muy relevantes en el resultado final. As, realizar el cross validate con el valor por defecto y para el modelo final bajar este valor a 2, que a mi entender es el que mejores recomendaciones devuelve.

    Hasta ahora hemos estado intentado validar como de buenos son los resultados en funcin del RMSE, sin embargo, esta mtrica no es adecuada para el problema que tenemos entre manos.

    Una mejor mtrica para ver como de bueno es el modelo es utilizar una mtrica de ranking. En este caso emplear NDCG (Normalized Discounted Cumulative Gain). Esta mtrica tiene en cuenta cuantos de los documentos recomendados por el modelo se encuentra en la lista de documentos verdaderamente tiles (es importante que los documentos estn ordenados). Puesto que nosotros no disponemos de esta lista de validacin, puede ser razonable utilizar como referencia los cantantes/grupos que escucha cada usuario. As, tomaremos como set de recomendaciones ptimas la lista de reproduccin de cada usuario.

    Para poder utlizar la funcin ndcgAt de pyspark.mlib es necesario estructurar las recomendaciones de la siguiente manera:

    $$([\text{lista reproducciones ordenada para usuario 1}], [\text{recomendaciones ordenadas para usuario 1}]),[\text{lista reproducciones ordenada para usuario 2}], [\text{recomendaciones ordenadas para usuario 2}]),...$$
